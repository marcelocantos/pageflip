pageflip --help-agent
=====================

PURPOSE
  Capture a region of the screen on a timer and write a PNG whenever the
  contents change more than a configurable perceptual-hash threshold. The
  primary consumer is a Claude Code session that receives each new slide
  as it lands.

INVOCATION
  pageflip [--region X,Y,W,H] [--interval SECS] [--threshold N] [--output DIR]
  pageflip --version
  pageflip --help
  pageflip --help-agent

OPTIONS
  --region X,Y,W,H   Rectangle to capture. X,Y,W,H are in logical points
                     (the same coordinate space as macOS `screencapture -R`
                     and window-position APIs). X,Y is the top-left corner
                     in absolute virtual-desktop coordinates; W,H are width
                     and height. Captured PNGs are written at the display's
                     physical-pixel resolution, so on a 2x Retina display a
                     200x200 region yields a 400x400 PNG — full-resolution,
                     not blurred. Required until the interactive picker
                     (🎯T2) lands.
  --interval SECS    Seconds between captures. Floating-point. Default: 2.
  --threshold N      Minimum pHash Hamming distance from the last saved
                     frame required to save a new frame. Default: 10.
  --output DIR       Directory to write PNGs into. Default: a timestamped
                     directory under the current working directory.
  --version          Print version and exit.
  --help             Print human-oriented help and exit.
  --help-agent       Print this machine-oriented help and exit.

EXIT CODES
  0   Clean exit (including --version, --help, --help-agent, and a
      successful capture).
  1   Runtime error (permission denied, region off-screen, backend
      failure, PNG encode failure).
  2   clap-level argument parse error (wrong flag, malformed value).

OUTPUT CONTRACT
  One line is written to stdout per saved PNG, formatted as:
      <absolute-path>\n
  Downstream consumers (e.g. the T5 feeder) tail this stream.

LOOP BEHAVIOUR
  pageflip runs an interval loop: capture, hash with pHash (8x8 DCT +
  mean), compare to the last saved frame's hash, and save only when the
  Hamming distance ≥ --threshold. The first frame always saves. Tick
  cadence targets --interval; if a capture overruns, the next tick fires
  immediately. Ctrl-C (SIGINT) breaks the loop after the in-flight
  capture completes and exits 0.

PLATFORM STATUS
  macOS arm64   primary target (🎯T1.1, 🎯T1.2, 🎯T1.3 complete)
  Windows       planned (🎯T6)
  Linux X11     planned best-effort (🎯T7)

PERMISSIONS (macOS)
  Requires Screen Recording permission (System Settings > Privacy &
  Security > Screen Recording). On first run the OS prompt appears; if
  the user denies, pageflip exits with a readable error (🎯T1.2).
